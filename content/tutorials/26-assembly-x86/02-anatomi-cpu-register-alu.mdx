# Anatomi CPU: Register, ALU, dan Orkestra Silikon

Kita masuk ke dalam chip.

Bayangkan Anda menyusut hingga seukuran atom dan melangkah masuk ke dalam prosesor Intel Core i9 Anda. Di hadapan Anda bukan lautan sirkuit yang acak вҖ” melainkan sebuah **kota yang terorganisir secara luar biasa**, dengan jalur data yang bergerak pada kecepatan cahaya, gudang penyimpanan sementara yang sangat kecil namun sangat cepat, dan sebuah pabrik aritmetika yang tidak pernah berhenti bekerja.

Inilah dunia dalam CPU вҖ” dan Assembly adalah bahasa yang Anda gunakan untuk **memerintah kota ini**.

## 2. Register: Tangan Kerja CPU

Register adalah **variabel yang tersimpan langsung di dalam chip CPU**, bukan di RAM. Kecepatan aksesnya hampir **instan** вҖ” mengakses register ~1 clock cycle, sementara mengakses RAM bisa membutuhkan 100-300 clock cycles.

Dalam arsitektur x86-64 modern, ada **16 register general-purpose** utama:

> [!IMPORTANT] Backward Compatibility yang Brilian
> Ini adalah warisan dari sejarah 46 tahun! Ketika Intel merilis chip 64-bit, mereka tidak membuang register 32-bit (EAX) atau 16-bit (AX). Mereka hanya **meluaskan** register yang sama. `RAX` adalah versi 64-bit dari `EAX`, yang merupakan versi 32-bit dari `AX`. Satu register, empat nama, empat ukuran berbeda.

### 2.2 Register Khusus yang Tidak Boleh Sembarangan Disentuh

```nasm
; Register Khusus:
RIP  ; Instruction Pointer - menunjuk ke instruksi BERIKUTNYA yang akan dieksekusi
RFLAGS ; Menyimpan hasil perbandingan (Zero Flag, Carry Flag, Sign Flag, dll)
```

**RIP (Instruction Pointer)** adalah register paling kritis. Ia selalu menunjuk ke lokasi memori di mana instruksi Assembly berikutnya berada. Ketika CPU selesai mengeksekusi satu instruksi, RIP otomatis melompat ke instruksi berikutnya.

> [!CAUTION] RIP adalah Mahkota Kejahatan
> Hampir semua teknik exploit tingkat lanjut (buffer overflow, ROP chains) bertujuan untuk **mengontrol register RIP**. Jika seorang penyerang bisa mengarahkan RIP ke kode jahat miliknya, ia menguasai program Anda sepenuhnya. Inilah mengapa pemahaman Assembly sangat penting dalam keamanan siber.

### 2.3 RFLAGS: Register yang Menyimpan "Perasaan" CPU

Setelah setiap operasi matematika atau perbandingan, CPU menyimpan "hasilnya" di register RFLAGS:

```
RFLAGS Register (64-bit):
Bit 0:  CF (Carry Flag)     вҶ’ Set jika ada overflow dari unsigned addition
Bit 2:  PF (Parity Flag)    вҶ’ Set jika hasil memiliki jumlah bit genap
Bit 4:  AF (Adjust Flag)    вҶ’ Digunakan untuk BCD arithmetic
Bit 6:  ZF (Zero Flag)      вҶ’ Set jika hasil operasi = 0 (sangat penting!)
Bit 7:  SF (Sign Flag)      вҶ’ Set jika hasil negatif (bit tertinggi = 1)
Bit 8:  TF (Trap Flag)      вҶ’ Untuk debugging (single-step mode)
Bit 9:  IF (Interrupt Flag)  вҶ’ Mengaktifkan/menonaktifkan hardware interrupt
Bit 11: OF (Overflow Flag)  вҶ’ Set jika ada overflow dari signed operation
```

Flags ini adalah yang membuat `if/else` bisa berjalan di Assembly! Ketika Anda menulis:
```c
// C code
if (a == b) { ... }
```

Kompiler menterjemahkannya menjadi:
```nasm
; Assembly equivalent
cmp rax, rbx    ; Bandingkan RAX dengan RBX (RAX - RBX, tapi hanya set flags)
je  label_true  ; Jump ke label_true HANYA JIKA Zero Flag = 1 (artinya sama!)
; ... kode jika tidak sama ...
jmp label_end
label_true:
; ... kode jika sama ...
label_end:
```

## 3. ALU: Pabrik Aritmetika dan Logika

**Arithmetic Logic Unit (ALU)** adalah jantung perhitungan CPU. Ia hanya bisa melakukan dua jenis operasi:

### Operasi Aritmetika
```nasm
add rax, rbx    ; RAX = RAX + RBX
sub rax, rbx    ; RAX = RAX - RBX
mul rbx         ; RDX:RAX = RAX * RBX (hasil 128-bit!)
div rbx         ; RAX = RDX:RAX / RBX, RDX = sisa bagi
imul rax, rbx   ; RAX = RAX * RBX (signed multiplication)
```

### Operasi Logika (Bitwise)
```nasm
and rax, rbx    ; RAX = RAX AND RBX  (per-bit)
or  rax, rbx    ; RAX = RAX OR RBX
xor rax, rbx    ; RAX = RAX XOR RBX (trik: xor rax,rax = set RAX ke 0!)
not rax         ; RAX = NOT RAX (flip semua bit)
shl rax, 1      ; RAX = RAX << 1 (shift left = multiply by 2)
shr rax, 1      ; RAX = RAX >> 1 (shift right = divide by 2)
```

> [!TIP] Trik XOR untuk Zero
> `xor rax, rax` adalah cara paling efisien untuk mengset register ke 0 di x86. Kenapa? Karena `X XOR X = 0` secara matematis, dan instruksi ini menghasilkan opcode yang lebih kecil (2 byte) dibanding `mov rax, 0` (7 byte). Kompiler C selalu menggunakan trik ini!


### Contoh Nyata: `add rax, rbx`

Mari ikuti satu instruksi Assembly dari awal sampai akhir:

**Fase 1: FETCH**
```
RIP = 0x400100  вҶ’ CPU membaca bytes di alamat memori 0x400100
Bytes yang dibaca: 48 01 D8  (ini adalah opcode untuk "add rax, rbx")
RIP sekarang maju ke 0x400103 (instruksi selanjutnya)
```

**Fase 2: DECODE**
```
Control Unit membaca 48 01 D8:
48    = REX prefix (menandakan operasi 64-bit)
01    = ADD opcode (tambahkan register ke register)
D8    = ModRM byte (tentukan operan: RBX ke RAX)
```

**Fase 3: EXECUTE**
```
ALU mengambil nilai dari RAX dan RBX
ALU melakukan penjumlahan
RFLAGS diupdate (ZF, CF, OF, dll)
```

**Fase 4: WRITEBACK**
```
Hasil penjumlahan disimpan kembali ke register RAX
```

## 6. Koneksi Fisik: Bagaimana CPU Berbicara ke Dunia Luar

CPU tidak bekerja sendirian. Ia terhubung ke dunia luar melalui sistem bus:

```
CPU в”Җв”Җ[Front Side Bus / QPI / PCIe]в”Җв”Җв–ә Memory Controller в”Җв”Җв–ә RAM
CPU в”Җв”Җ[DMI / PCIe]в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв–ә PCH Chipset в”Җв”Җв”Җв”Җв”Җв”Җв–ә HDD/SSD
CPU в”Җв”Җ[PCIe x16]в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв–ә GPU
CPU в”Җв”Җ[PCIe / DMI]в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв–ә USB Controller в”Җв”Җв–ә Keyboard/Mouse
```

Dalam Assembly, Anda berkomunikasi dengan perangkat ini melalui dua mekanisme:
1. **Memory-mapped I/O**: Perangkat "terlihat" sebagai alamat memori
2. **Port I/O**: Instruksi khusus `in` dan `out` untuk berkomunikasi dengan port
