## Kata Pengantar: Android Sebenarnya Adalah Linux!

Siapa sangka, di balik cantiknya ikon warna-warni *WhatsApp* dan Game *Mobile Legends*... ternyata *Smartphone* yang menempel setiap detik di tangan Anda tak lain dan tak bukan adalah sebuah **Mesin Super Komputer Mini berbasis LINUX (!)**. 

Betul sekali, *Android* hanyalah sebuah baju, sedangkan otot yang menarik urat nadi mesinnya adalah Kernel Linux tulen berdarah murni (Mirip dengan OS Ubuntu pada Laptop Hacker/Server). 

Inilah proses sesaat setelah Jendral Gerbang Utama (Bootloader) musnah dan membangkitkan roh sakti **The Kernel**.

## 1. Momen Penyerahan Mahkota (Kernel Wakes Up)

Sebelum lenyap, Aboot/Little Kernel "memukul keras" titik saraf memori SoC di alamat RAM spesifik, membangunkan kepingan Kernel (partisi *boot.img*). 

Momen pertama Kernel membuka matanya disebut **"Kernel Initialization"**. Pada 100 milidetik pertama, The Kernel melakukan invasi besar-besaran:
- **Pendataan Populasi (Hardware Probe):** Ia memeriksa setiap mili blok SoC. Mengecek apakah Chip GPU sehat, Layar Touchscreen terpasang, Kamera Sony siap diakses, dan Sensor Sinyal merespons panggilannya.
- **Sistem Syaraf (Memory Subsystem):** Mulai memetakan dan menjajah alokasi Kapasitas RAM. Memisahkan `zRAM` khusus swap, dsb.

Tanpa detak jantung Kernel, *Motherboard* hanyalah serpihan pasir logam tanpa nyawa.

## 2. Sang Mandor Pertama Kehidupan: "Proses `Init` (PID 1)"

Setelah kernel terbangun dan merasa kuat, ia membentangkan sebuah rahim ajaib untuk melahirkan mahluk Hidup/Program pertama kalinya di atas singgasana Mesin ini.

*Bayi Program* yang pertama (memiliki kode *Personal ID 1 / PID 1*), adalah Mbah Buyut dari seluruh aplikasi di muka bumi Android Anda. Namanya tak lain adalah **"The Init Process"**. Jika **Init** mati, Kernel Linux akan lumpuh terkena *Kernel Panic* (mati total / seketika reboot HP-nya!).

> **Pola Pikir (Mental Logic):**
> Anda bisa menebang (Force Close) aplikasi TikTok (Misal dia punya KTP *PID Nomor 5323*). Anda bisa paksa tutup Google Chrome (*PID 4212*). Semua tak apa. Namun, bila Anda menjadi *Root Hacker* dan dengan sengaja membunuh *"Sang Kakek Buyut Murni"* `kill -9 1` (Membunuh aplikasi bernomor tanda pengenal 1), Sistem HP akan musnah dan HP langsung *Restart* karena nyawa alam semesta utama telah diputus.

The Init inilah yang lalu menancapkan tongkat estafet dan berteriak *"Jalankan Semuanya!!!"*. Ia membaca file partisi biner bernama `.rc files` (seperti `init.rc`). 
File script magis ini adalah *Surat Wasiat Pabrik* yang memberitahu sang Buyut Init urutan perakitan rumah:
*"Hai Init, tolong pasangkan partisi System, aktifkan getaran Haptic, nyalakan layar terang, dan hidupkan robot Java bernama Zygote!"*

## 3. Modifikasi Taraf Dewa (Custom Kernel / Oprek)

Sering mendengar *"Install Custom Kernel kaya Franco, ElementalX, Hydra, Agni"* di forum _XDA Developers_?

Seorang Pakar Opreker (*Reverse Engineer*) menolak memakai Kernel bawaan pabrik. Kenapa? Karena Pabrik (Misal Samsung/Xiaomi) merancang baterai agar awet sehingga CPU di batasi tenaganya 70%.

Hacker mengambil Kode Sumber *Source Code Linux C++* tersebut, mengubah script agar CPU berjalan melebihi batas 110% (*Overclocking*), mengatur warna *Display Control* menjadi gila tajam (*K-Lapse*), dan menurunkan setrum asupan (Undervolting) agar HP bisa melempar panas. 

Setelah diedit, Kernel ditutup rapat, dimampatkan menjadi satu file **`boot.img`** kembali. Lalu disuntikkan secara brutal ke Kavling Partisi Boot menggunakan jurus sakti Fastboot Mac/Windows Terminal yang kita pelajari di bab sebelumnya! 

> **Skema Sihirnya:**
> _PC Hacker (Linux Terminal) -> Fastboot Modus -> Kabel USB -> UFS Partisi Boot -> Jebol -> "Selamat, Anda memakai Mesin Balap F1!"_

<Callout type="info">
**Fenomena Layar Bootloop 💀**
Pernahkah Anda mengutak-atik HP dan terhenti hanya "Muter-muter Logo Samsung melotot sejam" tanpa henti? Itulah fenomena menyeramkan The Bootloop Maut!
Secara logik *Reverse Engineer*, Bootloop diakibatkan ketika Sang Kernel nyala dengan selamat, namun Mbah Buyut System Utama (Proses Init PID 1) gagal merangkai bata / membaca file (*Corrupted*) saat membangun kota aplikasinya. Jadi Sang Kakek Buyut bengong dan me-_refresh_ proses membaca layarnya terus menerus, tak pernah bisa menekan bel pintu *"Home Screen"* Anda!
</Callout>
