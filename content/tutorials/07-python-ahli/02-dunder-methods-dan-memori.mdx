## 1. Dunder Methods (Metode Sihir Bergaris Bawah Ganda)

Sering bertanya dari mana datangnya lambang aneh `__init__` saat kita di bab kelas cetakan OOP? Mengapa depannya ada dua garis bawah `__` dan belakangnya `__`?

Metode beridentitas garis bawah kembar itu dinamakan ***Dunder (Double Under)*** atau **Magic Methods**. Mereka bukanlah pabrik/fungsi biasa. Mereka adalah JANTUNG BAWAAN refleks/otomatis dari Python. Jika Anda mengubah kode di dalam rahim mesin dunder ini, Anda sedang **meretas (*hacking*)** tabiat default Python itu sendiri!

```python
class BarangKargo:
    
    # KEDATANGAN ROH AWAL: Dipanggil paksa otomatis sepersekian 
    # milidetik ketika jiwa pertama Object menyentuh dunia RAM!
    def __init__(self, nama, berat):
        self.nama = nama
        self.berat = berat

    # MESIN HACK KODE TEKS: Apa yang terjadi saat programmer 
    # kolot nekat mencoba mem-print() langsung badan fisik Class Anda ke terminal?
    # Jika Anda tidak mengatur ini, Python akan teriak ngeri men-print (Object Address 0x0FA2..)
    def __str__(self):
        # Kita sulap layar agar keluar kata rapi!
        return f"Ini adalah fisik box: {self.nama} dengan bobot seberat {self.berat} KG."

    # MESIN HACK OPERATOR TAMBAH(+): Mari kita rekayasa agar 
    # 2 Kargo Ajaib bisa dicampur tambah dengan tombol operasi Matematika (+) !
    def __add__(self, kargo_kawan):
        # Saat Kargo A + Kargo B, berat dan gelarnya otomatis lebur!
        return BarangKargo(self.nama + " Plus " + kargo_kawan.nama, self.berat + kargo_kawan.berat)

# ==========================================

benda_1 = BarangKargo("Berlian", 5)
benda_2 = BarangKargo("Baja", 100)

# Keajaiban Hack Dunder __str__ : Print benda yang bukan tipe data dasar!
print(benda_1)      
# Hasil: "Ini adalah fisik box: Berlian dengan bobot seberat 5 KG."

# Keajaiban Hack Dunder __add__ : Cukup tekan Plus tanpa rumus aneh!
mutasi_gabungan = benda_1 + benda_2   
print(mutasi_gabungan.nama)  
# Hasil: "Berlian Plus Baja" (Menjadi mutan!)
```
Inilah mengapa alat `List` atau `Math` di Python bisa begitu indah, karena di dalamnya, Bapak Kreator Python menggunakan trik Dunder untuk menyembunyikan rumitnya logika *hacking* core ini.

## 2. Sang Algojo Pembersih: Garbage Collector

Anda terus bermain membuat Stiker Nama Variabel dan mencetak Object kue *Class* ke memori RAM jutaan kali. Suatu ketika, Anda menimpa/membuang koneksi stiker tersebut.
*(Contoh: Data stiker mantan pacar yang lama, ditimpa oleh nama Data stiker pacar baru)*.

Lalu, ke mana jasad data mantan ini pergi di dimensi RAM? Apakah ia diam mejajah memori menjadi hantu yang bocor (Memory Leak)?

Tidak! Berbahagialah kita di dunia Python. Tidak usah seperti orang kuno C++ yang harus memanggil mesin `Delete()` untuk menghancurkan mayat memori; Python telah menerjunkan pasukan maut senyap rahasia bernama **<Glossary definition="Engine/Mesin peronda otomatis yang jalan di belakang layar (background) terus menerus bertugas mencari data zombie yang Stiker/Label Tali pengikat variabelnya telah terputus ke objek apapun. Data zombie ini akan langsung DIBUNUH dan dihancurkan dari sirkukasi RAM.">Garbage Collector (Pengumpul Sampah)</Glossary>**.

### Konsep Tali Pengikat (Reference Counting)

Setiap wujud data `Object` *(layaknya Balon helium)* punya **Berapa BANYAK tali stiker (Reference) yang menggenggamnya kuat di bumi**. 

```python
import sys # Sistem perpustakaan jantung mesin OS

# 1. Balon Helium Angka List Kosong ini dipegang oleh Tali (A)
tali_A = [1, 2, 3]

# 2. Oh, Tali (B) juga ikut memegangi balon ujung kardus yang persis SAMA!
tali_B = tali_A

# Sistem Mengecek: Hei Balon [1,2,3], berapa jumlah total orang yang merantai tali ke tubuhmu?
print(sys.getrefcount(tali_A)) 
# Jawabnya akan berkisar di "3" (A + B + Tali Ghaib sys pengecek sendiri)

# ================================

# MASA DEPAN KEMATIAN...
# Kita putus dan bakar Tali A (Menjadi list baru atau Tipe NULL/None)
tali_A = None 

# Kita putus dan bakar juga Tali B
tali_B = None

# SESAAT ITU JUGA SAAT SEMUA TALI MENJADI ANGKA 0 ...
# SANG ALGOJO KEMATIAN (Garbage Collector) LANGSUNG DATANG DI DETIK ITU, 
# MENEMBAK BALON [1,2,3] DAN MENGHANCURKANNYA LEBUR JADI ANGIN DI RAM MEMORI!
```

Mental logic ini membuat Anda sadar di tingkat *Mega-Expert*. Kalau Anda salah menaruh *"Tali Global"* tak terlihat yang mengingat Database 2 Juta baris milik pelanggan Anda, Algojo Garbage Collector **tidak akan pernah berani membunuh Data Raksasa itu** karena merasa "Wah, masih ada tuan Master Variabel yang pegang tali ini!".
Alhasil? Server Web Anda akan **BOCOR MELEDAK MEMORI (Macet / Out of Memory OOM Error)**. 

Jadi, bijak-bijaklah melepas Tali `Variabel / Array` (misal dengan mengubahnya menjadi nilai Kosong `None` atau lenyapnya Indentasi batas fungsi) saat Anda sudah tak memerlukannya di server mahal 1 Triliun Anda.
