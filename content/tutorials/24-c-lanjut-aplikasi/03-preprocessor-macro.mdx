# Preprocessor Macro: Sang Pengendali Compiler

Apakah Anda menyadari bahwa garis kode seperti `#include <stdio.h>` itu memiliki simbol pagar (`#`) yang janggal dibanding sintaks C yang normal? Pagar itu bukanlah bagian murni dari tata bahasa C!

Itu adalah bahasa rahasia dari **The Preprocessor**, sebuah agen pengawas program pembaca teks kilat yang me-‘razia’ seluruh file `.c` Anda tepat detik-detik pertama proses kompilasi sebelum The C Compiler terbangun membaca kodenya.

Jika C merombak memori di level arsitektural saat aplikasi *berjalan (runtime)*, maka Preprocessor merombak teks sintaks murni C itu sendiri sebelum aplikasinya *tercipta (Compile time)*. Mari bedah kontrol mutlak ini.

## 1. Menempa Substitusi Teks Mutlak (`#define`)

Hal terbaik untuk menghindari angka keras (*Hardcoded Magic Numbers*) yang bertebaran liar membingungkan mata manusia pada file raksasa adalah dengan memaksa Preprocessor melakukan skema temukan-dan-ganti (Find and Replace).

```c
#include <stdio.h>

// MACRO CONSTANT (Selalu HURUF KAPITAL)
#define KAPASITAS_MAKSIMAL 500
#define NAMA_SERVER "THE_TITAN_SERVER_X"

int main() {
    int pemain_aktif = 120;
    
    // Saat Compiler C membaca baris bawah ini, ia DITIPU!
    // Compiler tidak melihat tulisan 'KAPASITAS_MAKSIMAL', 
    // ia melihat murni tulisan '500' karena sebelum compiler tiba preprocessor sudah menyalin-tindihnya di belakang layar.
    
    if (pemain_aktif > KAPASITAS_MAKSIMAL) {
        printf("KAPASITAS SERVER %s MELEDAk!\n", NAMA_SERVER);
    }
    
    return 0;
}
```

> [!CAUTION] Bencana Kutipan Macro (Macro Functions)
> The Preprocessor sangat bodoh. Ia mensubstitusi apa pun sesuai huruf pas bandrol mentah-mentah. Jangan pernah merancang the *Macro Function* sok pintar seperti `#define KUADRAT(x) x * x`. Jika dipanggil dari koding `KUADRAT(3 + 1)`, preprocessor akan membentangkannya sintaks aslinya menjadi `3 + 1 * 3 + 1` hasilnya = 7! Bukannya 16!
> **Aturan Sakral Ekstrim**: Sediakan tameng KURUNG tutup di setiap parameter Macro Anda: `#define KUADRAT(x) ((x) * (x))` !!

## 2. Pagar Ghaib Conditional Compilation (`#ifdef`)

Apakah Anda pernah bingung, bagaimana cara pembuat The Linux Node.js Runtime atau NginX Web Server bisa menulis source code C tunggal mereka secara sakti lintas platform sehingga berjalan di Microsoft Windows dan sejalan bersama the Linux OS? Walau The Windows memakai fungsi aneh `Sleep()` sedangkan di Linux OS adalah fungsi kecil `sleep()`.

Jawabannya adalah **Conditional Compilation (Kompilasi Bersyarat)**.
Anda memerintahkan the Preprocessor dari skema C untuk menendang baris blok *tidak sesuai target platform* dan menurunkannya membuangnya dari teks layar mentah-mentah C seolah-olah blok Windows itu tak pernah tertulis di dalam source di Linux!

```c
#include <stdio.h>

// The Preprocessor dapat mencium rahasia lingkungan THE OS KITA:
#ifdef _WIN32
    // Blok ini HANYA akan disisakan dan selamat jika anda mem-build-nya lewat OS Windows
    #include <windows.h>
    #define BERSIHKAN_LAYAR "cls"
    #define SALAM "Halo dari Bil Gates!"
#else
    // Blok ini akan bertahan jika anda Build menggunakan OS Linux/Mac
    #include <unistd.h>
    #define BERSIHKAN_LAYAR "clear"
    #define SALAM "Halo dari Sang Pinguin Linux Torvalds!"
#endif

int main() {
    printf("%s\n", SALAM);
    
    // Panggil The OS raw system command 
    // secara fleksibel dari sistem yang benar saja!
    system(BERSIHKAN_LAYAR); 

    return 0;
}
```

## 3. The Include Guard

Ini skenario mengerikan The Architecture Systems: Bayangkan file `player.h` memasukkan The include file `senjata.h`, namun diam-diam file `musuh.h` rupanya di ujung dunia lain ikut memasukkan `senjata.h`. Ketika the Preprocessor merajut ketiga header ke atas dalam `main.c`, file The `senjata.h` tercetak dua kali berulang membuat mesin bentrok *Multyplty Declared Symbol*.

Sumbatan mutlak dari semua kekacauan ini dinamakan **Include Guard (\#pragma once)**.

Gaya Klasik The C90:
```c
// File: senjata.h
#ifndef SENJATA_H   // "Jika the SENJATA_H belum diciptakan?"
#define SENJATA_H   // "Ciptakan sekarang di The Cache!"

struct Senjata { int atk_dmg; };

#endif 
// Berkat tameng pagar di atas, masuk 100 kalipun The Preprocessor tidak akan menyalin isi tengahnya kembar.
```

Gaya Modern C11 (Tersupport Modern Clang/Gcc Compiler):
```c
#pragma once // Murni satu baris pengaman ganda! Sangat cepat. Bebas bentrok.

struct Senjata { int atk_dmg; };
```
